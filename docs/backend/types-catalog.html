Types catalog

Shared types in lib/types. If you understand these, everything else in the backend flows naturally.

APIControl

APIControl enumerates target strings used in query-driven endpoints. 

export namespace APIControl {
  export namespace Project {
    export namespace Get {
      export enum Target { ALL, ALL_AS_LIST, MY }
      export enum Portfolio { ANY, GAME, GRAPHICS, RND }
    }
  }
}

Service data

Service inputs and outputs are namespaced by feature. Inputs live in SDIn (ServiceDataIn), outputs in SDOut (ServiceDataOut). 

export namespace SDIn {
  export namespace Blog {
    export type Create = {
      title: string;
      slug: string;
      content: string;
      tags: string[];
      coverImgUrl: string | null;
    };
  }
}

export namespace SDOut {
  export namespace Blog {
    export type GetAsList = { _id: string; title: string }[];
  }
}

Service signatures

Every service follows the same function signature and returns a discriminated union. 

export type ServiceSignature =
  (data: SDIn, session: RequireSession extends true ? ISession : ISession | null)
    => Promise>;

export type IServiceResolve =
  | { success: true; data: SDOut }
  | { success: false; errorCode: ESECs; errorMessage?: string };

Error codes (ESECs)

ESECs is an enum of service error codes. The E prefix denotes enums in this codebase. 

export enum ESECs {
  USER_NOT_FOUND,
  INVALID_CREDENTIALS,
  FORBIDDEN,
  PROJECT_NOT_FOUND,
  BLOG_NOT_FOUND,
  MEDIA_NOT_FOUND,
}
