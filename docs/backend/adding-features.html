Adding features

Repeatable steps for new endpoints and new domain features.

How to add a new API endpoint

  1. Define the input schema in lib/validators.
  2. Add input/output types in lib/types if needed.
  3. Implement the service function in lib/services.
  4. Create the route handler under app/api.
  5. Update docs and the system map.
Example route setup

import createHandler from "@/lib/handler";
import widgetValidator from "@/lib/validators/widget.validator";
import widgetService from "@/lib/services/widget.service";

const POST = createHandler({
  requireAuth: true,
  validationSchema: widgetValidator.create,
  options: {
    service: widgetService.create,
  },
});

export { POST };

How to add a new backend feature

  1. Create a Mongoose model in lib/database/models.
  2. Add a repository in lib/database/repos.
  3. Wire service logic in lib/services.

    4. Coding conventions

    • Always use createHandler for route handlers.
    • Validate every request with Zod; add shared fields to core.validator.
    • Prefer shared types in lib/types; keep only truly local types inline.
    • Services must call repositories, not raw Mongoose models.
    • Return ESECs codes for expected errors; throw AppError for unexpected failures.
    • Avoid the deprecated email/password sign-up flow.
