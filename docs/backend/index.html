This guide explains how the backend works, how data flows through the API layer, and how to extend the system without surprises.

Last updated: 2026-03-13

System snapshot

  • API routes live under next/src/app/api.
  • Requests flow through createHandler for validation and auth.
  • Services encapsulate business logic and call repository classes.
  • Mongoose models define User, Team, Blog, Project, Media, and Featured.
  • Authentication is JWT-in-cookie with Google OAuth as the primary flow.

Roles and access

Role Typical access
GUEST Read-only access to public data.
MEMBER Create and manage own projects, blogs, and media.
ADMIN Manage users, teams, featured content, and deletions.
ROOT Reserved for future high-privilege actions.
Email/password sign-up exists but is deprecated. Google OAuth is the supported onboarding path.
