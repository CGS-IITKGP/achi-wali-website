What each area does

  • app/api: Next.js route handlers for every endpoint.
  • lib/handler.ts: Shared request pipeline and response shaping.
  • lib/services: Business logic and authorization rules.
  • lib/database: Mongoose connection, models, and repositories.
  • lib/validators: Zod schemas for request validation.
  • lib/utils: Helpers for env vars, logging, email, cookies, and Cloudinary.
  • proxy.ts: Middleware guardrails for auth, dashboard, and admin routes.
  • docs/backend: Static documentation site and system map.
