Authentication flow

How Google OAuth creates a session and how JWT cookies are used.

Google OAuth flow

  1. Client hits /api/auth/google to start OAuth.
  2. User authenticates with Google and is redirected back.
  3. /api/auth/google/callback exchanges the code for an ID token.
  4. Server verifies the ID token and creates or fetches a user.
  5. A session JWT cookie is set and the user is redirected.

Session lifecycle

  • JWTs are signed with BACKEND_JWT_SECRET_KEY.
  • Protected routes check the cookie via authService.extractSession.
  • /api/auth/refresh-session rotates the token.
  • /api/auth/sign-out overwrites the session cookie.
Email/password sign-up exists for legacy users but is deprecated. New accounts should use Google OAuth.
